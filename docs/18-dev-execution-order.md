开发顺序清单

1. M1-基础搭建：项目结构、环境变量、数据库连接、迁移工具
   完成标准：能本地启动后端+前端空页面
2. M2-用户与鉴权：/auth/register /auth/login + token 中间件 + 角色守卫
   完成标准：游客/用户/管理员权限生效
3. M3-风格任务链路：POST /tasks/style-transfer + GET /tasks/{id} + 状态流转（单用户串行，不引入任务队列）
   完成标准：任务可从 pending/running 到 success/failed，且同一时刻仅 1 个推理任务执行
4. M4-作品入库与大厅：POST /artworks、GET /hall/feed、GET /artworks/{id}
   完成标准：作品能发布到大厅并展示详情
5. M5-互动功能：点赞、评论、关注（含防重复与自关注校验）
   完成标准：计数正确、冲突返回 40901
6. M6-管理员能力：管理作品/评论/用户 + admin_logs
   完成标准：管理操作可审计可追踪
7. M7-前端联调：用户端4页+管理员端核心页
   完成标准：主流程全走通（注册->生成->发布->互动->管理）
8. M8-测试与答辩材料：接口测试清单、异常用例、部署说明、演示脚本
   完成标准：可稳定演示一遍完整业务流程
